# 添加上游账号余额查询

## Goal

让管理员在账号管理界面按需查询 `AccountTypeUpstream` 上游账号的剩余额度，以便判断充值、启用或切换账号的时机，同时不影响正常代理链路。

## Background

- 本地 `Channel` 不持有上游地址、密钥或唯一账号引用；实际连接配置由 `AccountTypeUpstream` 账号持有，因此查询目标确定为账号而不是渠道。
- 首批兼容平台为 sub2api 和 New API。`cc-switch` 的 New API 实现表明，用户余额接口可能需要区别于推理 API Key 的访问令牌和用户 ID。
- 多个本地账号可能属于同一上游用户。本次不查询用户全部分组或套餐倍率，避免重复请求及不明确的去重语义。

## Requirements

### R1. 查询目标与入口

- 账号管理表格增加独立的“账号余额”列。
- 只有 `AccountTypeUpstream` 账号提供余额查询；其他账号显示不适用并继续沿用现有用量展示。
- 每个可查询账号在余额旁提供单账号刷新按钮；页面顶部提供“刷新当前页账号余额”按钮。
- 已禁用的上游账号仍可查询，查询不得启用账号或修改健康、调度及代理状态。

### R2. 平台与凭证配置

- 上游账号可选择 `自动识别`（默认）、`sub2api` 或 `New API`。
- 明确选择平台时只使用对应协议；自动识别只在账号配置的同源地址上进行有限兼容探测，不根据域名猜测平台，也不跨域发送凭证。
- 查询优先复用账号已有的上游地址和推理凭证。
- 管理员可选填仅供余额查询使用的管理访问令牌和用户 ID；专用凭证不得参与正常代理请求，也不得在普通账号响应、日志或错误信息中泄露。
- 现有推理凭证权限不足时，必须提示需要配置余额查询凭证，不得显示为零余额。

### R3. 余额响应语义

- 至少支持 sub2api 与 New API 的已验证余额协议；无法识别的平台返回明确的“不支持”结果。
- 查询结果必须区分余额口径：`user` 显示“用户余额”，`api_key` 显示“Key 额度”，无法可靠判断时以 `unknown` 显示“可用额度”。
- 结果包含余额、单位和查询时间；平台能够从同一次余额响应可靠返回当前推理 API Key 倍率时可同时显示该单一倍率。
- 不得为了倍率额外请求管理接口，不得用用户分组倍率推断当前 Key 倍率，也不得返回空的用户分组倍率占位数据。
- 上游超时、网络错误、鉴权失败和异常响应必须产生可理解且可区分的反馈，不能影响正常代理请求。

### R4. 当前页批量刷新

- 批量范围严格限定为当前分页内的 `AccountTypeUpstream` 账号，包括禁用账号。
- 批量请求采用受限并发；同一账号已有查询进行中时不得创建重复请求。
- 每行独立更新，单个失败不能中止其他账号，也不能清除其他行已经成功的结果。
- 批量完成后显示顶部汇总通知，分别统计成功、失败和不支持数量，同时保留逐行状态。

### R5. 状态与安全

- 余额及查询状态仅保存在当前页面内存，不写入账号记录或其他数据库表；重新进入页面后显示“未查询”。
- 成功结果显示查询时间。刷新失败时可以保留同一页面会话内最近一次成功结果，但必须同时显示本次失败状态。
- 只有管理员可调用查询接口或读取结果。
- 日志、响应和错误不得包含完整 API Key、访问令牌、认证头或未经清洗的上游响应体。

## Acceptance Criteria

- [ ] AC1：管理员可在“账号余额”列单独刷新任一启用或禁用的 `AccountTypeUpstream` 账号；非上游账号不会调用余额接口。
- [ ] AC2：管理员可选择自动识别、sub2api 或 New API；明确类型不探测其他协议，自动识别不跨域或无限探测。
- [ ] AC3：查询复用现有地址和可用凭证，并支持选填专用管理访问令牌及用户 ID；专用凭证不会进入普通代理链路或对外暴露。
- [ ] AC4：用户余额、当前 Key 额度和未知口径的可用额度具有不同标签；权限不足不会被显示成零余额。
- [ ] AC5：同一余额响应含有可靠的当前 Key 倍率时予以展示；否则不增加请求、不猜测、不显示默认倍率。
- [ ] AC6：超时、网络错误、鉴权失败、异常响应和不支持协议均有明确反馈，且查询前后账号状态、健康及调度信息不变。
- [ ] AC7：一键刷新只处理当前页上游账号，采用受限并发和请求去重；部分失败不影响其他行，结束后显示成功、失败和不支持汇总。
- [ ] AC8：余额只存在于页面内存，成功结果显示查询时间；重新进入页面后恢复“未查询”，数据库不保存余额。
- [ ] AC9：非管理员不能调用查询接口；自动化测试覆盖两平台解析、权限、错误、脱敏、单行状态和当前页批量行为。

## Out of Scope

- 上游用户全部分组或套餐倍率查询与去重。
- 为倍率发起额外管理请求。
- 自动充值、余额不足时自动停用或切换账号。
- 定时轮询、余额持久化、历史趋势和余额告警。
